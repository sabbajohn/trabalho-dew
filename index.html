HTML e Acessibilidade

Semântica, leitura estruturada e navegação por teclado

Semântica é o uso de elementos HTML com significado. Ela organiza o conteúdo, facilita a manutenção e melhora a interpretação da página por leitores de tela, navegadores e motores de busca.

O que é semântica e por que ela importa

  • Define claramente o papel de cada parte da página.
  • Melhora a hierarquia de leitura para leitores de tela.
  • Facilita manutenção porque cada bloco fica previsível.
  • Ajuda a separar conteúdo principal de conteúdo complementar.

Como melhorar acessibilidade no código

  • Usar títulos em ordem lógica e landmarks como header, nav e main.
  • Garantir contraste adequado, foco visível e textos compreensíveis.
  • Evitar controles falsos quando um link comum resolve o problema.
  • Usar medidas relativas para respeitar zoom e preferências do usuário.

Principais tags semânticas usadas ou exemplificadas

Cada card abaixo resume uma tag importante para organização do documento.

<header>

Representa o cabeçalho da página ou de uma seção.

<nav>

Indica a área de navegação com links para partes do site.

<main>

Delimita o conteúdo principal e único da página.

<section>

Agrupa um tema específico dentro do conteúdo.

<article>

Isola um bloco independente, como notícia, card ou texto próprio.

<aside>

Mostra conteúdo complementar ao fluxo principal.

<figure>

Agrupa uma mídia, ilustração ou exemplo visual relacionado.

<figcaption>

Fornece legenda contextual para a figura.

<time>

Marca datas e horários de forma compreensível para máquinas.

<mark>

Destaca trechos importantes sem perder o contexto do texto.

<footer>

Encapsula o rodapé de uma página ou de uma seção.

Navegação otimizada por teclado

Nesta página, a navegação foi simplificada com links reais e um skip link. Isso evita depender de controles artificiais e faz o percurso com a tecla Tab ficar previsível.

  • O primeiro foco útil é o link para pular direto ao conteúdo.
  • O menu principal usa âncoras sem JavaScript.
  • Os estados de foco usam :focus-visible.
  • As seções possuem id para navegação direta.

CSS e Layout

Medidas, Flexbox, Grid, Mobile First, responsividade e animações

CSS não serve apenas para "deixar bonito". Ele organiza o layout, controla comportamento em diferentes telas e ainda pode comunicar interação com transições e animações.

Medidas fixas e relativas

Escolher a unidade certa influencia legibilidade, adaptação e acessibilidade.

px

Medida fixa

Boa para bordas, sombras, ícones e pequenos detalhes que precisam de precisão.

border: 2px solid;
rem

Escala a partir da raiz

Excelente para tipografia e espaçamentos porque respeita o tamanho base do usuário.

padding: 1.5rem;
%

Proporcional ao pai

Ajuda a criar larguras flexíveis em grids, colunas e blocos fluídos.

width: 100%;
vw / vh

Relativas à viewport

Úteis em seções de destaque, respiros grandes e composições ligadas à tela.

min-height: 100vh;
clamp()

Escala controlada

Combina valor mínimo, fluido e máximo para evitar exageros em telas muito grandes.

font-size: clamp(1rem, 2vw, 1.2rem);

Quando usar cada tipo?

Fixas funcionam bem em detalhes visuais precisos. Relativas são melhores para textos, espaçamentos e blocos que precisam se adaptar a celulares, tablets e desktops sem quebrar a leitura.

Flexbox x Grid

Ambos são importantes, mas não resolvem exatamente o mesmo problema.

Flexbox

Ideal para distribuir elementos em uma dimensão: linha ou coluna. Nesta página, ele ajuda no menu, nos grupos de tags e em blocos que precisam alinhar itens rapidamente.

Menu Cards Tags

Grid

Ideal para estruturas mais amplas em duas dimensões. Ele foi aplicado nas grades de cards e nos agrupamentos de conteúdo que mudam de uma coluna para várias colunas conforme a largura da tela aumenta.

Responsividade e Mobile First

Responsividade é a capacidade do layout se adaptar a diferentes tamanhos de tela. Neste projeto, a base começa pequena, com uma coluna e foco em leitura. Depois, as media queries ampliam a composição para tablets e desktops.

  • Base otimizada para telas pequenas.
  • @media (min-width: 48rem) reorganiza colunas e grades.
  • @media (min-width: 75rem) amplia densidade e hierarquia visual.
  • Menu, cards e seções crescem sem gerar overflow horizontal.

Meta viewport e imagens responsivas

O uso da meta viewport garante que o navegador renderize a largura correta em dispositivos móveis. Já imagens e mídias devem respeitar o contêiner para não ultrapassar a tela. 

<meta name="viewport" content="width=device-width, initial-scale=1.0">

img {
  max-width: 100%;
  height: auto;
}
